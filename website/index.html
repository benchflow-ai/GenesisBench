The idea

Most robotics benchmarks fix the environment and test a policy. We invert that contract: we test whether a coding agent can build one.

In the spirit of post-training benchmarks for language models, each agent is given real resources — a physics simulator and compute nodes with H100s — and left to work: write the code, shape the rewards, train the policy. GenesisBench scores what the resulting policies actually achieve, in simulation and on robot hardware.

GenesisBench 1.0 asks how language intelligence improves physical intelligence. GenesisBench 2.0 extends the question to world intelligence.

  1. Provision

    Each agent gets a simulation environment — Genesis, Isaac Sim, RoboCasa, or MuJoCo — and H100 compute.

  2. Train

    The agent writes the code, shapes the rewards, and post-trains a control policy. No human in the loop.

  3. Evaluate

    Policies are scored on held-out tasks and perturbations — generalization, not memorization.

  4. Transfer

    The strongest policies run on real robot hardware to measure the sim-to-real gap.

Preliminary task ideas are previewed at benchflow.ai/genesis.
